Primitives

Primitives

Number, boolean, string, undefined, and null are primitives

    Number

  • Only one number type in JS (64-bit floating point)
  • Decimal number's arithmetic is inexact. 0.1 + 0.2 != 0.3. BUT 1 + 2 === 3
  • Invalid arithmetic operations or invalid conversions will not throw an exception, but will result in NaN "Not-a-Number"

    String

  • can be inside '' or "".
  • "text".substring(1,3) // "ex"
  • strings, like all primitives, are immutable. concat() doesn't modify the existing string but creates a new one.

    Boolean (true and false)

  • falsy values: false, null, undefined, ''(empty string), 0 and NaN are falsy.
  • truthy values: all other values, including objects are truthy
  • Falsy value is evaluated to false in a boolean context.
  • True AND False results in False

Primitives (except null and undefined) are treated like objects, in the sense they have methods but they are not objects...

Numbers, strings, adn booleans have object equivalent wrappers (Number, string, and Boolean functions)...

in order to allow access to properties on primitives, JS creates an wrapper object and then destroys it. (optimized by JS engine).

Primitives are immutable and objects are mutable

Variables

Can't use variables before declaring them now (in ES5 we could)

1 == true = true (becomes JS performs type coercion

1 === true = false (no type coercion)

"2" + 3 returns "23" (because 3 is coerced to "3" and the strings are added

    Let

  • Can be mutated(same thing as var)
  • Dont' have to assign them right away

    Const

  • can't mutate by assigning
  • BOTH let and const are block-scoped (function scoped in ES5)
  • HAVE to assign const variables right away

    'this' keyword

  • Not a primitive data type
  • It is created for each execution context
  • In a method call, the 'this' keyword poitns to the object that it's attached to
  • In a method call, the 'this' keyword is only defined as soon as the method gets called
  • 'this' inside a function POINTS TO THE Global Object (even inner functions)

    Hoistings

  • Variables are set to undefined in the creation phase
  • Hoisting variables in JS are available BEFORE the execution actually starts
  • Benefit of Hoisting is being able to use functions before defining

Variables associated with objects DON'T contain the objects, they contain a reference (pointer)

Variables associated with primitives (strings, numbers) actually HOLD the data

Objects

dynamic collection of properties...key = unique string (coverted to a string if non-string used)...value = primitive, object, or function

Variables associated with objects DON'T contain the objects, they contain a reference (pointer)

Variables associated with primitives (strings, numbers) actually HOLD the data

A JS function is an object becaus its prototype is an object, and so it inherits methods from the prototype

Execution object
  • stores variables
  • Global execution Object: Code that isn't in any function...Window object in browser
  • any function call is a new execution object
  • Hoisting variables in JS are available BEFORE the execution actually starts. (don't have to declare functions() before calling it!)
  • Benefit of Hoisting is being able to use functions before defining.
  • ONLY way to create a new scope in JS is to create new functions. lexical scoping a function() inside another functions gets access to the scope of the outer functions()
